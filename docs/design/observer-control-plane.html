System Drift

Health and benchmark motion

14:05 post-care +13
The observer tracks runtime drift against the startup baseline and preserves benchmark state before and after self-care.
1. Services emit structured signals
Gateway, Trigger, cheap runtime, and pipeline managers publish small health and event summaries.
2. Pipeline managers own pipelines
Detailed pipeline and agent activity is written into agent.db.
3. Observer aggregates system state
System-wide health, drift, and remediation history are written into system.db.
4. Self-care runs bounded actions
Only allowed cleanup and restart-request behaviors run inside the self-care budget.
5. Dashboard reads observer state
The dashboard does not scrape raw service internals by default.

Rule

Never overwrite the pre-care check. Preserve all three views so the observer can detect improvement, failure, or regression.

system.db

Control-plane truth

  • service registry and current up/down state
  • observer events and benchmark snapshots
  • self-care cycles and remediation history
  • dashboard and report query surface
agent.db

Pipeline detail

  • pipeline manager state and queue depth
  • agent events and task-level history
  • session traces and detailed error records
  • not the observer’s default data source
